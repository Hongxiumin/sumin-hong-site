# 이미지 기능 추가 업데이트 안내

## 무엇이 추가되었나요?

### 1. 이미지를 넣을 수 있는 5곳
- **프로필 사진** (Hero 영역, 이름 옆 원형)
- **경력 로고** (학교/직장/학회 아이콘, 각 경력 항목 왼쪽)
- **연구 시스템 대표 이미지** (TeacherGen@i, ETHOBOT 등 카드 상단)
- **강의/특강 사진** (정규 강의, 초청 강연 각 항목 옆)
- **갤러리 섹션** (홈페이지 하단, 여러 장의 사진 그리드)
- **특강 사진 스트립** (최근 특강 문구 아래 가로 스트립)

### 2. 자동 이미지 압축
- 관리자 페이지에서 이미지 업로드 시 자동으로 리사이즈되어 저장됩니다.
- 아이콘/로고는 PNG(투명 배경 지원), 사진은 JPEG로 저장됩니다.
- 프로필: 400×400 · 로고: 120×120 · 연구 이미지: 800×450 · 강의 사진: 400×300 · 갤러리: 1200×900

### 3. 라이트박스
- 갤러리와 특강 스트립 사진을 클릭하면 큰 화면으로 볼 수 있습니다 (ESC로 닫기).

### 4. 파일 크기 표시
- 관리자 페이지 상단에 현재 content.json의 예상 크기가 실시간으로 표시됩니다.
- 2MB를 넘어가면 노란색 경고로 바뀝니다. (GitHub Pages는 100MB까지 허용하지만, 로딩 속도를 위해 2MB 이하 권장)

## 배포 방법 (GitHub)

### 옵션 A: 전체 파일 교체 (권장)
1. GitHub 저장소(https://github.com/hongxiumin/sumin-hong-site)에 접속
2. `index.html`, `admin.html`, `content.json` 세 파일을 새 파일로 교체
   - 각 파일 클릭 → 연필 아이콘(Edit) → 내용 전체 복사/붙여넣기 → Commit
   - 또는 "Add file → Upload files"로 세 파일 한 번에 드래그앤드롭
3. 몇 분 후 사이트에 반영됨

### 옵션 B: HTML만 먼저 교체 (안전)
현재 `content.json`은 그대로 두고 `index.html`, `admin.html`만 먼저 교체해도 정상 동작합니다.
사이트가 잘 뜨는지 확인 후, 관리자 페이지에서 이미지를 추가하고 다운로드 → 그때 `content.json` 교체.

## 관리자 페이지에서 이미지 추가하는 방법

1. `https://hongxiumin.github.io/sumin-hong-site/admin.html` 접속
2. 원하는 항목에서 **"이미지 선택"** 버튼 클릭
3. 파일 선택 → 자동으로 리사이즈 & 미리보기 표시
4. 다른 항목도 추가/수정 후 상단 **"content.json 다운로드"** 클릭
5. 다운로드된 `content.json`을 GitHub 저장소의 기존 `content.json`에 덮어써서 커밋

## 참고 사항

- **하위 호환**: 이미지가 없는 항목은 예전과 똑같이 표시됩니다.
- **갤러리 섹션**: 사진이 하나도 없으면 자동으로 숨겨집니다. 첫 사진을 추가하면 나타납니다.
- **네비게이션**: 갤러리 섹션이 활성화되면 상단 메뉴에 "Gallery/갤러리" 항목이 자동으로 추가됩니다.
- **파일 크기 팁**: 원본 크기가 크더라도 자동 압축되므로 걱정 없이 올리시면 됩니다. 다만 갤러리에 사진이 많아지면 (예: 20장 이상) 로딩이 느려질 수 있어요.
